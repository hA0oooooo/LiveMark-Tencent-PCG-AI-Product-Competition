# LiveMark

LiveMark 是一个面向小红书电竞比赛直拍 KOC 的 AI 素材管理与内容复盘 Demo。它关注到了直拍创作者在“素材太多、难以判断、发布后难以复盘”的具体问题，帮助用户把线下活动中拍到的视频素材转化为可分析、可发布、可复盘、可沉淀的账号资产

### 业务定位

* 目标用户是小红书电竞比赛直拍 KOC，尤其是围绕选手互动、舞台现场、直播未覆盖视角、人物表情和粉丝情绪价值进行内容创作的普通创作者

* 核心问题是创作者往往拥有大量现场素材，但缺少稳定的判断框架：不知道哪几秒更值得发布，不知道应该用什么标题和封面包装，也很难从发布后的浏览、点赞、收藏和评论数据中总结出下一次拍摄和发布的经验

* LiveMark 的产品流程包括账号画像维护、视频上传、关键帧抽取、素材分析、候选片段建议、发布矩阵生成、发布后数据录入、内容实验复盘和长期记忆更新

* 在素材分析阶段，系统会结合关键帧画面、活动背景、目标人物和场景类型，分析画面中的互动强度、情绪价值、稀缺视角、封面潜力、清晰度和潜在风险，并生成候选片段建议与“涨粉机会评分”

* 在发布阶段，系统会生成小红书发布矩阵，包括标题、封面文案、正文、标签、评论引导和推荐发布时间，帮助用户围绕同一条素材形成多个可比较的发布方案

* 在复盘阶段，用户手动录入浏览、点赞、收藏、评论等数据，系统会对比原始素材特征、发布策略和最终表现，生成复盘报告，并提出下一条内容建议、下一场活动拍摄重点和账号长期记忆更新建议

### 技术实现

* 前端使用 Next.js、TypeScript、Tailwind CSS 和 Recharts，主要实现 Dashboard、账号基准、素材库、视频分析页、发布矩阵和复盘中心

* 后端使用 FastAPI、SQLAlchemy 和 SQLite，按照 routes、services、repositories、models、schemas 分层组织，分别负责 API 路由、业务逻辑、数据访问、数据库模型和接口数据结构

* 视频处理部分使用 ffmpeg 对上传视频进行抽帧，并使用 OpenCV 计算关键帧的 Laplacian variance 作为清晰度指标，结合时间间隔和画面质量选择代表性关键帧

* AI 分析部分通过 DashScope OpenAI-compatible API 调用 Qwen 多模态模型和文本模型。多模态模型用于理解关键帧中的人物、场景、互动和情绪价值；文本模型用于生成发布矩阵、复盘报告和长期记忆更新建议

* Agent 设计采用固定工作流，而不是完全开放式自主 Agent。系统会按照“素材分析 → 发布建议 → 数据复盘 → 记忆更新”的链路运行，并把账号画像、账号基准、历史内容、素材分析结果、发布计划和复盘结果写入数据库，作为下一轮判断的上下文

* 部署方式采用腾讯云轻量应用服务器一体部署。Nginx 对外提供 80 端口，`/api` 请求代理到 FastAPI 服务，页面请求代理到 Next.js 服务，服务进程由 PM2 或 systemd 管理。当前 Demo 使用 SQLite 作为轻量级数据库，后续可以迁移到 PostgreSQL 以支持多用户和更复杂的数据分析
